Project standard

Paper first. Data second. Writing last.

The workflow is deliberately strict: identify the original paper's logic, map the exact local data objects that can support replication, compute from the executable dataset stack, and publish a paper rather than a memo.

Step 1

Read the source study

Theory, estimand, sample logic, and every major decision point are extracted before any coding starts.

Step 2

Map local evidence

Only current executable local datasets are used, with each required empirical object tied to a real file.

Step 3

Write the full paper

The final output is a journal-style manuscript with results, derivations, decisions, and references.

Publishing workflow

How a new replication reaches the website

01

Source reading and decision mapping

The original paper is read in detail, and the required design decisions, constructs, and estimands are written down first.

02

Executable local reconstruction

The paper is rebuilt only from processed local datasets that already exist in the workspace, with explicit compromises when exact original objects are unavailable.

03

Long-form manuscript build

The evidence is expanded into a 30 to 40 page journal-style replication paper with detailed analysis, derivations, and comparisons.

04

GitHub Pages refresh

The PDF archive, JSON library feed, README, and public homepage are regenerated together so the website always reflects the published repository state.
